# 2026-05-09

## Fixed
- 修复拖拽标签到第一个位置时插入指示线不显示的问题（为 .header-tabs 容器添加左侧 padding）
- 修复 Rich 编辑模式下多行选区工具栏错误贴到后续行的问题，改为始终锚定选区起始行，并兼容全选时的文档起点定位。
- 调整 Rich 与 Source 编辑模式的全选工具栏行为，统一固定显示在选区下方，且不再因视口边界做翻转或收敛。
- 修复 Rich 与 Source 编辑模式全选工具栏的定位偏移问题，改为基于完整选区矩形计算锚点，而不是基于起始 caret 点。
- 修正选区工具栏下方优先策略，不再按“全选”判断，而是当锚点滚出当前可视区顶部时优先显示在下方。
- 调整选区工具栏的下方回退逻辑：当顶部无法显示时，改为基于完整 selection 底部定位，而不是基于首行锚点定位。
- 修复 Source 模式尾部空行导致工具栏落到未高亮空白区下方的问题，并将 Rich 模式的上下翻转判断改为基于编辑区可视顶边而非浏览器窗口顶边。
- 修复 Rich 模式尾部空行未被高亮时仍参与 selection 底部定位的问题，下方回退时会忽略末尾换行，仅基于实际高亮内容计算位置。
- 调整 Rich 模式工具栏下方定位实现，改为直接基于选区首尾坐标组装矩形，不再依赖 posToDOMRect，避免块级边界把工具栏误拖到未高亮空白区下方。
- 重构 Rich 模式选区工具栏定位方案：移除 BubbleMenu 依赖，改为与 Source 模式一致的绝对定位宿主，并由 rich adapter 直接提供首行锚点与实际高亮选区矩形。
- 调整 Rich 与 Source 选区工具栏的垂直回退规则：当顶部放不下时，始终贴 selection 底部显示，不再因视口底部不足把工具栏强行拉回视图中部。
- 进一步收紧 Rich 与 Source 选区工具栏的边界规则：若上方和下方两种位置都不在当前视图内，则将工具栏固定在当前视图底边，而不是裁切到视图顶边。
- 修复 Rich 模式选区工具栏未跟随外层 BScrollbar 滚动重算的位置陈旧问题，滚动编辑容器时会同步刷新 toolbar 浮层位置。
- 调整 Rich 模式选区工具栏的下方回退语义：顶部放不下时始终贴 selection 底部，不再触发“固定到视图底边”的特殊分支。
- 移除 Rich 模式 Markdown 序列化中人为注入的 `&nbsp;` 占位符，避免导出内容出现 HTML 实体；连续空行现按普通 Markdown 语义收敛。
