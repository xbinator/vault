# 2026-04-19

## Changed
- 为 Electron 下载配置 `npmmirror` 镜像源，避免默认官方源超时导致开发环境安装不完整。
- 统一 Electron 开发与构建脚本为 `pnpm run`，避免在 pnpm 项目中混用 `npm run` 导致启动链路不一致。
- 在 `package.json` 中补充 pnpm 原生依赖白名单，允许 `electron` 与 `better-sqlite3` 执行安装脚本，修复开发环境缺失 Electron 二进制的问题。
- 修复 BEditor 源码模式 CodeMirror 高度样式，确保内容高度交给外层滚动容器计算。
- 修复 BEditor 源码模式目录锚点无法跳转的问题，通过 CodeMirror 文档位置计算驱动外层滚动容器。
- 修复 BEditor 源码模式点击目录时同时触发 DOM 锚点和 CodeMirror 锚点滚动导致位置偏移的问题。
- 修复 BEditor 源码模式长文档后段目录跳转依赖 CodeMirror 估算高度导致滚动位置偏移的问题。
- 修复 BEditor 源码模式将 fenced code block 内伪标题误计入锚点编号，导致左侧目录跳转错位的问题。
- 修复 BEditor 源码模式滚动时目录高亮依赖 CodeMirror 可见行 DOM 的问题。
- 为 BEditor 源码模式补齐文内搜索能力，支持关键字高亮、上下匹配跳转和搜索状态同步。
- 调整 BEditor 源码模式文本选区样式，统一复用 `--selection-color` 与 `--selection-bg` 变量。
- 调整 BEditor 源码模式 Markdown 高亮主题色，匹配 Claude Light/Dark 配色。
- 为 BEditor 源码模式新增 fenced code block 语法高亮，复用 lowlight 语言词法结果映射到 CodeMirror decorations。
- 升级 `package.json` 中一批同 major 的前端与编辑器依赖版本，包含 AI SDK、Tiptap、Vite、Vue、UnoCSS 等，以降低升级风险并保持依赖新鲜度。
- 在保留现有 ESLint 与 Stylelint 版本不变的前提下，继续升级 Electron、TypeScript、Node 类型、`better-sqlite3`、`marked`、`vue-tsc` 等其余依赖。
- 兼容 TypeScript 6 对 `baseUrl` 的弃用校验，在根 `tsconfig` 中补充 `ignoreDeprecations` 配置以保持现有路径别名可继续编译。
- 将 Vue scoped 样式中的深度选择器改为 `:global(...)` 写法，避免 Lightning CSS 构建阶段保留 `deep` 语法并触发告警。
- 优化编辑器打包体积：将 Mermaid 预览改为按需加载，并将富文本编辑器与源码编辑器子面板拆为异步组件，降低编辑器入口 chunk 体积。
- 将 Vite 8 构建分包配置从已弃用的 `build.rollupOptions.output.manualChunks` 迁移到 `build.rolldownOptions.output.codeSplitting.groups`，与当前构建器能力保持一致。
- 修复开发环境启动地址对 `localhost` 解析的依赖，统一改为 `127.0.0.1`，避免当前环境下 Vite 与 Electron 开发启动失败。
- 补充 `electron/tsconfig.json` 的 `ignoreDeprecations` 配置，消除 Electron 主进程 watch 编译在 TypeScript 6 下的 `baseUrl` 弃用报错。

## Features
- 为 BEditor 源码模式新增 Markdown 符号高亮功能，支持标题标记、强调符号、代码标记、链接语法、列表标记、引用标记、表格语法、任务列表、转义字符等符号的可视化高亮。
- 调整 Markdown 符号高亮配色对比度，Light 主题颜色加深、Dark 主题颜色提亮，提升符号可读性。

- 实现 AI Tools Task 02：新增 active editor context registry 与内置只读工具，支持读取当前文档、读取当前选区和搜索当前文档，并补充对应 focused tests。

- 实现 AI Tools Task 03：扩展 BEditor 公共读写 API，统一暴露选区读取、光标插入、替换选区和替换全文能力，并在编辑器页面注册 active editor context。

- 实现 AI Tools Task 04：新增写工具确认适配层与内置写工具，支持光标插入、替换选区和危险级全文替换，并补充取消与无选区等 focused tests。

- 推进 AI Tools Task 05 的基础能力：新增 renderer 侧工具流转 helper，并在 useChat 中接入 tool-call 事件监听，为聊天侧工具执行循环做准备。

- 补充 docs/ai-tools 当前实现进度文档，并在 Tasks/README.md 标注 Task 完成状态，记录已完成的 Task 01-04 和进行中的 Task 05。

- 实现 AI Tools Task 05 主链路：在 BChat/BChatSidebar 中接入工具集合、确认适配器与 editor context，支持 tool-call 执行、tool result 回注和第二轮模型续流，并新增工具状态提示与重复调用防护。
- 收紧 AI Tools 聊天侧默认开放范围，仅默认暴露只读工具与 `insert_at_cursor`，暂不默认开放替换类工具。
- 新增 AI Tools provider capability gating，只有首批验证通过的 provider 会进入工具调用链路，未验证 provider 自动降级为普通对话。
- 补充 AI Tools focused tests，覆盖 provider 工具能力策略与默认工具集暴露策略，并完成 build 与 Electron main build 验证。
- 更新 docs/ai-tools 进度与 Task 文档，记录本轮已落地的 gating、默认工具集与验证结果。
