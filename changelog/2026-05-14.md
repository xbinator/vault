# 2026-05-14

## Changed
- 为 AI 工具流新增 `tool-input-start/delta/end` 前端透传，并在 `write_file` 最终 tool-call 到达前展示目标路径与内容流式预览，减少长文写入前的空白等待。
- 欢迎页、最近文件和原生打开入口现在会优先复用已打开标签；若目标磁盘文件未打开，则重新从磁盘读取，不再恢复旧草稿。
- 修复 `ask_user_question` 等待用户输入时未显示交互卡片的问题，同时兼容历史 `ask_user_choice` 消息在气泡渲染、消息提交和压缩保留中的识别。
- 统一测试与注释中的默认工具命名为 `ask_user_question`，并保留针对历史 `ask_user_choice` 会话的兼容性校验。
- 调整 `ask_user_question` 的等待态消息状态：释放全局提交流程，但保留助手消息的 `loading` 与未完成状态，避免交互阶段被误显示为已结束。
- 调整 `read_current_document`：未保存文档在无真实路径时会返回 `unsaved://{id}/{fileName}.{ext}` 形式的虚拟路径，默认扩展名为 `md`。
- 新增统一的 `unsavedPath` 工具模块，收口未保存文档虚拟路径的构建、判断与解析，并替换 `DocumentTool`、文件引用插入和未保存文件读取等分散实现。
- 为 `AIToolContext.document` 新增可选 `locator` 字段，显式区分真实磁盘路径与工具可消费定位符；`read_current_document` 与 `read_file(documentId)` 现已优先使用该定位符处理未保存文档。
- 调整聊天气泡工具栏显示规则：仅包含系统错误片段的助手消息不再显示复制与重试按钮，避免工具循环停止提示被误当作可重试回答。
- 调整工具循环停止提示的落点：当当前 assistant message 可复用时，不再新建独立错误消息，而是把终止原因追加到该消息的 `parts` 中，并以复用后的消息作为完成结果。
- 简化 `read_file` 输入模型：移除 `documentId` 旁路解析能力，现仅支持显式 `path` 读取，避免参数语义分叉。
- 调整 `write_file` 的路径解析与写入通道：现支持向 `unsaved://{id}/{fileName}.{ext}` 未保存草稿路径写入，命中当前编辑器草稿时直接替换文档，否则回写最近文件存储中的对应草稿内容。
- 收紧 `write_file` 的边界处理：统一错误消息提取逻辑，并为未保存草稿读写 helper 增加空 `fileId` 防御校验。
- 调整 `ask_user_question` 等待态的状态拆分：等待用户回答时输入区继续保持 loading/停止态，但 assistant message 本身结束当前流式展示，且仍允许提交用户选择继续续轮。
- 简化 `ask_user_question` 的用户输入模型：所有提问默认支持自定义文本输入，移除 `allowOther` 条件字段与相关分支判断。
