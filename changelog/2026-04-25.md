# 2026-04-25

## Added
- 新增 SQLite 引用链路回归测试，覆盖 `chat_messages.references_json` 的迁移、写入与读取。
- 新增聊天文本气泡的文件引用渲染测试，覆盖 user bubble 的 file chip 显示与 assistant 文本降级路径。
- 新增基于 `snapshotId` 的模型上下文构建测试，覆盖小文档全文注入与缺失快照回退行为。

## Changed
- 为 `chat_messages` 补充 `references_json` 列与迁移逻辑，确保 `references` 元数据在 SQLite 中真实落盘。
- 补齐聊天消息 SQLite 读写映射，将 `references` 纳入查询、UPSERT 与消息恢复流程。
- 调整 `MessageBubblePartText`，让用户消息中的 `{{file-ref:...}}` 稳定 token 渲染为只读 inline chip，并在缺失映射时安全回退为原始文本。
- 调整聊天提交流程，保留用户可见正文中的 file chip/token，同时在模型侧按引用快照补充全文或局部片段上下文。
- 新增 `chat_reference_snapshots` 持久化链路，在发送时为当前文档生成快照并按 `snapshotId` 构建模型上下文。
- 优化 `SelectionToolbar` 的“插入对话”行为，在点击工具栏按钮后继续保留富文本编辑器的选区与高亮，避免 Selection 视觉上消失。
- 优化 `BPromptEditor` 的文件引用插入逻辑，失焦后仍会恢复上一次有效光标位置，避免引用被错误插入到输入框起始处。
- 将 `BPromptEditor` 的光标缓存升级为基于节点路径与偏移的快照恢复，避免浏览器失焦后 `Range` 失效导致文件引用回退到开头插入。
- 前移 `BPromptEditor` 的光标缓存时机到输入、键盘移动与鼠标点击后，避免仅依赖 `selectionchange` 导致缓存长期停留在根节点 offset。
- 调整聊天引用插入顺序为“先显式记录输入框位置，再聚焦并按该位置插入”，避免失焦或侧边栏切换阶段污染 `BPromptEditor` 的最后光标位置。
- 修复 `BPromptEditor` 在文件引用插入后未刷新光标缓存的问题，确保连续多次插入会沿用最新插入点而不是第一次的位置。
- 收敛聊天输入框文件引用插入接口，移除重复的跨组件焦点恢复入口，改由 `BPromptEditor.insertFileReference` 内部统一负责恢复插入点与执行插入。
- 调整 `PaneRichEditor` 的“插入对话”选区保留策略，仅保留装饰高亮而不恢复真实选区，避免编辑器重新获取焦点后原生选中态残留不消失。
- 调整 `SelectionToolbar` 的 `BubbleMenu` 隐藏回调，在工具栏消失时同步关闭选区保留态，确保 Selection 视觉状态与浮层显隐保持一致。
