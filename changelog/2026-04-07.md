# 2026-04-07

## Added
- 新增数学公式（LaTeX）支持，基于 @tiptap/extension-mathematics 和 KaTeX
- 支持行内公式和块级公式渲染
- 添加 KaTeX 样式导入
- 在 Tauri 应用启动时打印数据目录路径
- 新增 `src/services/ai` 运行时骨架，包含类式 provider 注册中心、模型解析器和统一 `AIService` 入口
- 新增 AI 错误类型与错误归一化工具，统一配合项目 `asyncTo` 处理服务层异常

## Changed
- 修改 package.json 添加 @tiptap/extension-mathematics、katex 和 @types/katex 依赖
- 修改 src/assets/styles/index.less 导入 KaTeX 样式
- 修改 src/components/BEditor/hooks/useExtensions.ts 添加 Mathematics 扩展
- 修改 src-tauri/src/lib.rs 添加数据目录路径打印
- 修改服务商详情页，将连通性检查接入 `aiService.testConnection`
- 修改服务商 API 配置组件，支持选择启用模型并显示连通性检测状态
- 修改 `src/services/ai` 设计，移除抽象基类，改为 provider 接口加独立 helper 组合
- 修改 AI 错误处理职责，将 SDK 错误转换下沉到各 provider 实现内部
- 修改 AI 连通性检测实现，抽出默认 `testConnection` 逻辑供各 provider 复用
- 修改 AI 错误归一化逻辑，细化 OpenAI、Anthropic、Google 的鉴权、限流、模型和服务可用性错误映射
- 修改 `AIService` 返回约定，服务层内部封装 `asyncTo`，对业务统一返回 `[error, result]`
- 修改 `useProviders` 状态管理，提升为模块级共享单例，统一列表页、详情页、侧边栏和弹窗的数据源
- 修改服务商详情页模型管理逻辑，补全创建/编辑模型后的持久化与刷新同步链路
- 修改模型列表分类筛选逻辑，使“全部 / 对话 / 图片 / 视频”分段选项真正参与模型过滤
- 修改模型弹窗提交流程，将创建/编辑成功与失败提示收口到 `ModelModal` 内部，外层仅负责返回同步结果
- 修改模型保存职责划分，将模型创建/编辑的校验与持久化逻辑进一步迁入 `ModelModal`，外层仅在成功后刷新同步
- 修改模型启用/禁用职责划分，将切换与提示内聚到 `ModelList`，详情页仅在操作成功后刷新同步

## Features
- 支持 LaTeX 数学公式语法
- 行内公式使用 `$...$` 或 `$$...$$` 语法
- 块级公式使用 `$$$...$$$` 语法
- 公式渲染错误时不抛出异常，保持编辑器稳定性
- 应用启动时控制台输出 app_data_dir 和 app_local_data_dir 路径
