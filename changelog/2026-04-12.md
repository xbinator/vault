# 2026-04-12

## Added
- 引入 `electron-log` 日志库，将 Electron 原生 `console` 接管并配置为仅在控制台输出
- 主进程新增 `logger/service.mts` 和 `logger/ipc.mts`，并在 `electron/main/index.mts` 初始化
- 在 `types/electron-api.d.ts` 和 `electron/preload/index.mts` 暴露 `logger` 桥接 API
- 新增 `src/utils/logger.ts` 封装通用前端日志记录器
- 新增 `docs/code-wiki/` Code Wiki 文档，覆盖架构、模块职责、关键入口/收口点、依赖关系与运行方式

## Changed
- 替换 `ServiceConfig.vue` 和 `BModelIcon/index.vue` 中的原生 `console` 调用为新封装的 `logger`
- 完善 `AIInvokeResult` 类型，为 AI 文本生成（generateText）结果增加 token 消耗（usage）统计（包含 promptTokens、completionTokens、totalTokens）
- 将 AI 文本生成的 IPC 通道名由 `ai:generate` 修改为 `ai:invoke`
- 将前端预加载脚本中暴露的 API 由 `window.electronAPI.aiGenerate` 修改为 `window.electronAPI.aiInvoke`
- 修正 `src/shared/platform/electron-api.ts` 中对 `types/electron-api.d.ts` 的无效类型导出，保证 `tsc --noEmit` 通过
- 调整 `electron:dev` 启动脚本，使 `electron/` 编译产物变更后会自动重启 Electron 开发进程
- 将 Electron 开发环境的本地访问地址统一为 `127.0.0.1`，避免 `localhost` 解析异常影响启动
- 简化 `electron:dev` 启动等待条件，移除对前端开发地址的 `wait-on` 检查
- 拆分 Electron 开发脚本，新增 `electron:run` 以简化 `electron:dev` 的职责
- 优化 `BPromptEditor` 的输入安全性与编辑体验：HTML 转义防注入、纯文本粘贴、回车插入 `\n` 保持内容纯净
