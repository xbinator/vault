# 2026-04-03

## Added
- 编辑器新增“源代码编辑”模式，用于以纯文本方式编辑内容，并可通过“视图”工具栏在两种编辑模式间切换

## Changed
- TypeScript：tsconfig 增加 ignoreDeprecations 以通过 vue-tsc 类型检查
- 编辑器：将源码模式的标题与正文编辑能力收敛到 BEditor 内部，页面层仅保留模式切换和文件状态绑定
- 编辑器：将富文本视图与源码视图分别拆分为独立子组件，减少 BEditor 主组件模板与样式耦合
- 编辑器：将 RichEditorPane 的 Front Matter 交互改为 `v-model` 双向绑定，简化父子组件之间的数据同步
- 编辑器：将富文本与源码子组件中重复的标题输入框上提到 BEditor 顶层，统一标题展示与失焦处理
- 编辑器：抽离当前编辑器控制器，统一 `undo/redo/canUndo/canRedo` 调用，减少模式判断分支
- 编辑器：新增 `useEditorController` hook，下沉编辑器控制器适配逻辑，精简 BEditor 主组件
- 编辑器：源码模式正文输入框接入 `useTextareaAutosize`，按内容自动调整高度
- 编辑器：视图菜单新增“大纲”开关，可独立控制编辑器侧边大纲显示
- 编辑器：将视图状态收敛为单一 `viewState`，减少页面层分散控制
- 工具栏：移除独立选中值传参，改由 `ToolbarOption.selected` 直接驱动菜单勾选态
- 样式：移除 `:global(.dark)` 选择器，改用兼容 lightningcss 的暗色主题写法
- 编辑器：源码模式下禁用“大纲”菜单项，避免出现不可显示但仍选中的状态
- 编辑菜单：移除“复制全文”，新增“复制为纯文本 / Markdown / HTML 代码”
- 公共 Hook：新增 `useSafeClipboard`，统一封装剪贴板容错、空内容判断和复制提示
- 编辑菜单：新增“查找”操作，支持通过 `Ctrl+F` 在编辑器头部右侧显示内嵌查找输入框
- 编辑器：头部右侧查找框改为使用 `AInput`，统一输入控件风格
- 编辑器：移除 `useHoverIndicator` 及相关悬浮指示器接线，清理不再需要的悬浮提示逻辑
- 编辑器页面：补充 `ref` 导入，修复严格类型构建时报错
