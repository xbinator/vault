# 2026-04-16

## Changed
- 调整文件重新打开流程：已有最近文件记录时不再先用磁盘内容覆盖本地草稿，避免未显式保存的编辑内容在重新打开时丢失。
- 为最近文件记录补充磁盘同步基线，并在进入编辑器时比对草稿、上次保存内容和当前磁盘内容，只有在可确认安全或用户确认后才刷新为磁盘内容。
- 抽离文件内容冲突判定逻辑并补充自动化测试，覆盖草稿保留、磁盘刷新和已保存基线更新等关键分支。
- 新增独立 `test/` 目录集中管理自动化测试，避免测试文件分散在业务源码目录中。
- 为 `test/` 目录补充 README 约定，统一测试放置方式、命名规则和优先测试的抽象层级。
- 修复历史最近文件记录缺少 `savedContent` 时的兼容问题：重新打开含未保存草稿的本地文件时，会先用磁盘内容补齐保存基线，避免错误清除编辑器 dirty 星号。
- 为最近文件存储增加安全归一化迁移：无磁盘路径的历史草稿会自动补齐 `savedContent`，减少旧数据在后续打开时反复走兼容分支。
- 为聊天侧边栏新增会话与消息持久化基础设施：补充聊天类型定义、SQLite 表结构和聊天存储层，支持按会话类型查询历史消息。
- 解耦 `BChat` 组件的业务逻辑，将模型配置、流式请求和历史会话管理迁移到侧边栏容器，并将消息附件结构升级为可扩展的 `files` 列表与 `usage` 统计。
- 为聊天历史列表补充会话重命名能力。
- 将默认布局中的聊天侧边栏组件迁移到 `src/components/BChatSidebar/index.vue`，统一到项目现有的 `B` 前缀组件目录规范。
- 将聊天侧边栏中的历史会话列表拆分为 `src/components/BChatSidebar/SessionHistory.vue`，收敛父组件职责并统一目录内子组件组织。
- 将 `BChatSidebar` 的历史会话子组件进一步收拢到 `src/components/BChatSidebar/components/` 目录，统一复合组件的子模块层级。
- 将历史会话的重命名与删除交互内聚到 `SessionHistory` 子组件内部，减少 `BChatSidebar` 父组件的事件转发职责。
- 将历史会话展示改为下拉选择框形式，使用紧凑的会话选择器替代展开列表，并保留重命名与删除操作入口。
- 将历史会话选择器从 `BSelect` 调整为 `BDropdown` / `BDropdownMenu` 组合实现，统一到项目现有下拉菜单组件体系。
- 完善 `SessionHistory` 下拉弹层实现：修复模板结构问题，并补齐会话列表、空态、当前态高亮以及重命名/删除交互。
- 移除历史会话弹层中的重命名入口，仅保留会话切换与删除操作，收敛当前交互范围。
- 将历史会话列表改为按天分组展示，不再逐条显示具体时间，分组标题统一显示“今天 / 昨天 / 日期”。
- 将历史会话删除操作改为直接执行，不再弹出二次确认提示。
- 调整聊天会话生命周期：允许删除到 0 个会话并进入空态，点击“新建会话”只清空当前选择，真正发送消息时才创建新会话。
- 清理 `SessionHistory` 中删除按钮切换到 `BButton` 后残留的未使用样式选择器。
- 将 `BChat` 的消息双向绑定模型名从 `value` 调整为 `messages`，使组件接口语义更清晰。
- 重构聊天会话状态分层：`chatStore` 仅维护会话与消息列表，当前选中会话改由 `BChatSidebar` 本地状态管理，并补充按会话异步加载消息能力。
- 将当前块菜单的展开方向判断从浏览器视口改为基于编辑器定位容器的可用空间计算，避免嵌套容器或局部滚动场景下菜单方向错误。
