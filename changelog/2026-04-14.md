# 2026-04-14

## Added
- [Tabs] 创建 `src/stores/tabs.ts`，定义 `tabs` 和 `activeId` 状态。
- [Tabs] 提取并实现底层持久化辅助函数 `loadSavedData`, `persistData`。
- [Tabs] 配置 Pinia `$subscribe` 深度监听，实现多标签页数据的自动序列化持久化。
- [Tabs] 在 `tabs` store 中新增 `addTab` 和 `markDirty(id)` 方法。
- [Tabs] 创建 `src/components/BLayout/components/HeaderTabs.vue` 标签页 UI 组件，支持横向滚轮滚动。
- [Tabs] 在 `HeaderTabs.vue` 中处理文件名的格式化、点击切换高亮及关闭事件，并精准控制 `-webkit-app-region` 拖拽属性。
- [Layout] 在 `src/components/BLayout/index.vue` 中集成 `<HeaderTabs />`。

## Changed
- [Editor] 更新 `useDirty` Hook，改为通过文件 ID 追踪多个文件的脏状态（dirty state），并调用 `tabsStore.markDirty(id)` 同步状态。
- [Editor] 更新 `useFileActive` 的文件打开逻辑：不再直接覆盖 `fileState.value`，改为调用 `tabsStore.addTab()` 统一通过标签页管理多文件。
- [Editor] 移除 `index.vue` 中关于多文件状态的逻辑（移除隐含的 `tabsStore.activeTab` 监听器，改用 `computed` 响应式获取当前激活标签页的内容），并清理了原本的下拉菜单与最近搜索功能，实现编辑器与多文件状态管理的彻底解耦。
- [Provider] 将 `ProviderSidebar.vue` 内容迁移到 `src/views/settings/provider/layout.vue`，侧边栏改为布局内渲染。

## Removed
- 移除最近文件查询弹窗（`SearchRecent` 组件）及相关数据传递：`savedRecentFiles`、`handleSelectRecentFile` provide/inject、`visible.recentSearch`、`FILE_RECENT_MORE` 快捷键、工具栏「更多」入口。
- 移除查找功能：`FindBar` 组件、`EDIT_FIND` 快捷键、编辑菜单「查找」项（含 Electron 原生菜单）、`visible.find` 状态。
- 将 `editorInstance` 从 layout provide/inject 链中移除，改为在 `editor/index.vue` 本地声明模板 ref。
- 新建 `src/stores/editor.ts`，集中管理 `fileState`、`visible`、`sidebarState`、`viewState`、`editorInstance`。
- `layouts/default/index.vue` 移除全部 editor 本地状态声明（`fileState`/`visible`/`sidebarState`）和 4 个 `provide()` 调用，改为使用 `useEditorStore`。
- `views/editor/index.vue` 移除全部 `inject` 调用，改为直接使用 `useEditorStore`。
- `useViewActive`、`useEditActive` 直接从 store 读取所需状态，移除外部参数依赖。
