# 2026-04-22

## Added
- 新增 `BChat` 模型消息转换缓存能力，支持复用未变化的历史消息前缀。
- 新增聊天消息 `parts` 结构化片段，支持文本、思考、工具调用与工具结果的有序展示和持久化。
- 新增 `BChat` 结构化消息片段设计文档，记录 `parts` 写入、渲染、模型转换和持久化约定。

## Changed
- 为内置 AI 工具新增 `get_current_time` 只读能力，并同步加入聊天侧默认工具白名单与测试覆盖。
- 拆分内置工具的文档只读能力与环境只读能力，将时间工具从 `read.ts` 迁移到独立的环境工具模块。
- 整理内置工具工厂的组装逻辑，显式区分默认只读工具分组与可选写工具分组，降低后续扩展时的职责混淆。
- 抽取内置工具默认暴露名称清单，统一复用聊天侧默认工具策略，避免多处维护工具名称列表。
- 调整内置工具工厂基于共享清单筛选默认暴露工具，使 builtin 组装层与默认策略层统一由同一份 catalog 驱动。
- 将写工具确认从弹窗确认升级为聊天流内确认卡片，并新增会话级确认控制器、确认执行状态同步与过期处理规则。
- 为聊天流确认卡片补充展示层测试，并将确认卡片从 `MessageBubble.vue` 拆分为独立组件与展示逻辑辅助模块。
- 统一确认卡片的类名生成方式为共享 `createNamespace('message-bubble')`，并补充确认失败文案与默认折叠规则的展示层回归测试。
- 为确认卡片新增基于 `jsdom` 和 `@vue/test-utils` 的挂载测试，覆盖默认折叠策略、标题切换交互和确认动作事件发射。
- 为 `MessageBubble` 新增确认卡片集成挂载测试，覆盖消息流内渲染和确认动作向上透传。
- 将确认卡片样式从共享的 `message-bubble` 命名空间拆分为独立的 `confirmation-card` 命名空间，降低消息气泡与确认卡片之间的样式耦合。
- 将 `MessageBubble` 中的思考片段、工具调用片段和工具结果片段拆分为独立子组件，并抽出共享展示辅助函数，收紧主组件职责边界。
- 继续拆分 `MessageBubble` 的文本片段与附件头部组件，使主组件进一步收敛为消息分发与工具栏编排层。
- 拆分 `BChat` 服务配置读取与缺失配置时的 UI 提示逻辑，避免 `getServiceConfig` 混合业务和副作用。
- 提取 `resetToolLoopState()`，将请求中止时的运行时状态清理与新会话初始化解耦。
- 为 `BChat` 消息转换新增缓存回归测试，覆盖追加消息复用与历史消息变更失效两种场景。
- 调整 `BChat` 消息渲染与模型上下文转换逻辑，界面优先展示结构化 `parts`，`content` 保留为纯文本聚合内容。
- 为 `BChat` 深度思考片段补充折叠交互，并在工具调用入参为空时隐藏空白参数代码块。
- 调整 `BChat` 工具续轮的完成事件时机，确保中间轮次的工具调用和工具结果也会持久化到聊天记录。
- 为 `BChat` 工具结果片段补充折叠交互，并让工具续轮继续复用当前 assistant 消息，避免工具结果后拆成新的消息气泡。
- 调整 `MessageBubble` 工具结果片段的默认折叠策略，首次渲染时默认收起工具结果内容。
- 调整 `MessageBubble` 直接基于 `BBubble` 渲染，移除对 `BBubbleText` 的容器层依赖，收紧聊天气泡组件职责边界。
- 为 `BChat` 暴露输入框聚焦能力，并在聊天侧边栏点击新建会话后自动恢复输入焦点。
- 将聊天侧边栏当前激活会话 ID 下沉到 `setting` store 持久化，支持刷新后恢复最近会话。
- 调整聊天侧边栏头部布局，在左侧补充标题并保留右侧新建会话按钮。
- 将聊天侧边栏头部标题改为显示当前激活会话名称，空状态下回退为默认标题。

## Removed
- 无

## Features
- 优化长消息历史下的模型消息构建开销，减少工具续轮时重复转换。
