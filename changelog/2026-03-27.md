# 2026-03-27

## Added
- 为 BEditor 编辑器添加了粘贴功能支持
- 实现了代码块的现代化布局，包括头部语言标签和按钮区域
- 添加了代码高亮样式，使用 Atom One Dark 配色方案
- 为编辑器代码块新增复制按钮，支持一键复制代码内容
- 新增独立的 Toc 目录组件，并支持标题节点折叠/展开
- 为编辑器新增 Markdown 表格渲染支持

## Changed
- 封装了编辑器内容设置功能，创建了 `setEditorContent` 函数
- 优化了代码块的视觉效果和布局结构
- 修复了下拉菜单重复点击同一项时不会再次触发事件的问题，恢复“打开文件”等操作的重复可用性
- 将代码块渲染升级为自定义节点视图，统一展示语言标签、操作区和格式化代码内容
- 优化代码块语言标签展示格式，将语言名首字母改为大写
- 将代码块复制能力切换为基于 `@vueuse/core` 的 `useClipboard` 实现
- 优化目录导航点击跳转逻辑，改为驱动编辑器滚动容器精准定位到对应标题
- 优化代码块复制提示定时器，避免重复点击时反复创建多个 `setTimeout`
- 调整代码块视觉为接近 GitHub Light 的浅色风格，并同步更新语法高亮配色
- 修复点击代码块复制按钮时事件冒泡干扰编辑器，导致代码内容异常消失的问题
- 为代码块复制成功和失败补充全局 Toast 提示反馈
- 优化目录标题选择器的写法，避免超长硬编码字符串
- 移除目录项点击触发滚动的逻辑，目录仅保留展示与当前标题高亮能力
- 将目录数据改为基于 `content` 的计算属性，移除手动同步逻辑
- 将 Sidebar 重构为目录数据层，目录展示和折叠交互下沉到独立 Toc 组件
- 收敛目录相关组件粒度，将标题折叠树和类型定义合并回 Toc 组件内部
- 接入 TipTap 表格扩展，使 Markdown 表格内容可以正确解析和显示
- 补充表格包装层和单元格内容样式，修复表头下方表体内容显示异常的问题
- 调整 Markdown 内容加载链路，改为先转换为 HTML 再注入编辑器，修复复杂文档打开时报 schema 校验错误的问题
- 整理 BEditor 文件内容，收敛扩展注册、外部内容导入和表格兼容处理逻辑
- 在编辑页展示当前文件名，并统一同步浏览器标题与桌面窗口标题
- 补充 Markdown 无序列表样式，恢复 `ul/li` 的圆点、层级缩进和嵌套列表表现
- 调整 inline code mark 配置，支持 `**\`code\`**` 这类复合 Markdown 标记
- 为 Markdown 列表项解析增加段落兜底，修复复合标记出现在列表项中时触发 `Invalid content for node listItem` 的 schema 报错
- 将文件 API 从类实现重构为对象字面量实现，修复 `fileAPI.ts` 中 `max-classes-per-file`、`class-methods-use-this` 和 `no-return-await` 等 ESLint 报错
- 对齐 BEditor 自定义 Markdown 列表项解析的类型定义，改为复用 TipTap 官方类型，修复构建时 `parseMarkdown` 签名不兼容的 TypeScript 报错

## Features
- 支持在编辑器中粘贴 Markdown 内容
- 代码块支持语法高亮显示
- 代码块具有现代美观的布局设计
- 代码块支持显示语言信息并可直接复制内容
