# 2026-05-10

## Added
- 新增 InteractionContainer 交互容器组件，支持 Toast 提示和确认对话框
- 新增 ToastItem 组件，支持 error/warning/info/success 四种类型
- 新增 ToastStack 组件，管理 Toast 堆叠显示
- 新增 ConfirmModal 组件，支持模态确认对话框
- 新增 useInteraction hook，提供类型安全的交互 API
- 新增 useInteractionState hook，管理交互容器状态

## Changed
- VoiceInput 组件迁移到 useInteraction API，统一交互体验
- BChatSidebar 集成 InteractionContainer，提供统一的错误和确认交互

## Removed
- 移除内置工具 `search_current_document`，不再向 AI 暴露当前文档搜索能力。
- 移除编辑器态写工具 `insert_at_cursor`、`replace_selection` 与 `replace_document`。

## Changed
- 更新内置工具注册清单与相关测试，保持默认工具集合与工具续轮防护用例一致。
- 新增文件级写工具 `edit_file` 与 `write_file`，并要求基于 `read_file` 的完整读取快照执行修改。
- 统一内置文件工具源码命名为 `file-edit`、`file-read`、`file-write`，减少命名风格混杂。
- 新增 AI 文件工具协议设计文档，明确 `builtin`、共享文件状态、错误码与确认卡片的数据边界。
- 新增 AI 文件工具协议实现计划文档，拆分共享协议层、文件工具重构与确认卡片回归任务。
- 抽取 `src/ai/tools/shared` 文件协议层，新增共享快照、错误码与路径工具，并让 builtin 入口改用共享状态仓库。
- 为 `fileEdit` 与 `fileWrite` 新增模块内 `types.ts`，统一基于共享快照与错误助手执行文件写入校验。
- 调整确认卡片预览与状态文案，让 `edit_file` 和 `write_file` 使用一致的文件级展示语义。
