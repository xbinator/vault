# 2026-05-04

## Changed
- 为 `ConfirmationCard` 增加“自己输入”入口，提交时会取消当前确认并作为新的用户消息继续对话。
- 为 `update_settings` 确认卡接入自定义输入配置，允许用户在模型建议值之外继续补充自己的设置要求。
- 为 `insert_at_cursor` 确认卡接入自定义输入配置，允许用户在模型建议插入内容之外继续补充自己的文本。
- 为 `replace_selection` 确认卡接入自定义输入配置，允许用户在模型建议替换内容之外继续补充自己的文本。
- 为富文本编辑器的 Markdown 块节点补充真实源码行号映射，避免“插入对话”沿用渲染态文本行号。
- 将 `SelectionToolbar` 的文件引用行号切换为基于源码映射的真实 `startLine/endLine`，移除按 `textBetween()` 猜行号的旧逻辑。
- 将块级源码行号聚合细化为块内精确选区映射，修复多行段落中只选第 6 行却回退到第 5 行的问题。
- 为 `SelectionToolbar` 增加基于原始 Markdown 顶层 token 的源码行号回退映射，修复计划文档中因空行和未挂载源码 attrs 的块节点导致第 7 行回退为第 5 行的问题。
- 为聊天文件引用插入载荷补充 `renderStartLine` / `renderEndLine`，用于保留渲染视图中的用户感知行号。
- 扩展聊天文件引用 token 协议为 `{{#path start-end|renderStart-renderEnd}}`，并保持旧的 `{{#path start-end}}` 兼容可读。
- 为 Preload 层错误收集补充 `ResizeObserver` 噪音过滤，避免该类浏览器错误重复刷入日志文件。
- 调整 Preload 层错误日志首行格式，移除默认 `Error` 类型下重复的 `Error: Error:` 前缀，并补充对应回归测试。
- 新增“聊天语音运行时按需下载方案”设计文档，收敛首次按需下载与设置页管理入口的跨平台实现方向。
- 将语音设计方案调整为前端直接产出分段 `wav`，移除正式产品路径对 `ffmpeg` 的依赖，并补充大体积录音的分段与采样率策略。
- 为 Electron 主进程新增语音 runtime 管理与安装器基础层，支持读取安装状态、解析已安装 whisper 资源并预留按需下载入口。
- 将聊天语音录音链路切换为前端直接编码分段 `wav`，并在语音完成后把转写文本写回聊天输入框。
- 为设置页新增“语音组件”入口，支持查看语音运行时状态并提供下载、重装、删除操作。
- 新增 `docs/speech/README.md`，集中说明语音模块的整体方案、下载源设计、安装流程和当前代码落点。
- 新增 `resources/speech/manifest.json` 模板，并将程序读取的语音资源清单从说明文档中独立出来，默认使用 GitHub Release 分发 whisper 二进制、使用官方推荐地址分发 `ggml-base.bin`。
- 新增 `scripts/speech/manifest-tool.mjs` 与对应 npm script，用于生成语音资源 `sha256` 并校验 `resources/speech/manifest.json` 结构。
- 将语音 manifest 模板中的 GitHub Release 地址收敛为真实仓库 `xbinator/tibis`，并在 `docs/speech/README.md` 中补充 release 命名规范与发布回填清单。
- 为语音 manifest 工具新增 `fill` 命令，可根据本地三个 `whisper` 二进制和一个模型文件自动计算 `sha256` 并回填 `resources/speech/manifest.json`。
- 为语音 manifest 工具新增 `localize` 命令，可将 manifest 资源地址切换到本地静态服务，方便开发环境联调自动安装链路。
- 在根 `README.md` 中补充语音自动安装的本地开发文档，说明本地静态资源目录准备、manifest 本地化、`sha256` 回填和 `TIBIS_SPEECH_RUNTIME_MANIFEST_URL` 启动方式。
- 新增 `speech:dev:prepare` 与 `speech:dev:serve` 本地开发脚本，将语音自动安装联调收敛为“准备目录 + 启动静态服务”两条命令。
- 将本地语音联调目录统一收敛为仓库内 `.dev-resources/speech/`，同步更新 `.gitignore`、根 `README.md` 和 `docs/speech/README.md` 的开发说明。
- 在 `docs/speech/README.md` 中补充本地开发联调的 Mermaid 流程图，帮助理解从准备资源到点击麦克风触发安装与转写的完整链路。
- 将本地语音联调进一步收敛为固定源文件 `.dev-resources/speech/source/whisper-cli` 与 `.dev-resources/speech/source/ggml-base.bin`，并补充 `speech:dev:start` 统一启动静态服务和开发环境。
- 在根 `README.md` 与 `docs/speech/README.md` 中明确补充 `whisper-cli` 到 `.dev-resources/speech/source/whisper-cli` 的复制示例，避免本地联调时卡在固定文件名约定。
- 将本地开发语音源文件名统一为 `whisper-cli`，保持与 `whisper.cpp` 默认编译产物一致，减少手动重命名步骤。
