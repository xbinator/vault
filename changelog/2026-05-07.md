## Fixed
- 增量摘要窗口：`coordinator.ts` 新增 `resolveIncrementalWindow()`，增量模式下从 `coveredEndMessageId` 之后开始摘要
- 策略评估：`policy.ts` 新增 `buildEffectiveContextMessages()`，使 `charCount` 与 `assembler` 输出对齐
- 文件语义层：`planner.ts` 新增 `shouldPreserveAsFileSemantic()`，历史文件引用消息不再丢弃而是作为轻量语义输出
- 手动压缩降级：`compressSessionManually()` 在规则裁剪触发截断时自动降级为增量模式，记录 `degradeReason`
- 摘要有效性校验：`sqlite.ts` 读取时校验 `schema_version` 和 JSON 格式，无效记录自动标记为 `invalid`
- Summarizer 输入合约：`generateStructuredSummary()` 支持 `GenerateStructuredSummaryInput` 对象参数，增量模式下传入 `previousSummary`
- 修复 `degradeReason` 未持久化到数据库的问题：`sqlite.ts` 新增 `degrade_reason` 列的读写映射
- 修复文件语义消息与可压缩消息合并时顺序错乱的问题：按原始消息顺序排列以保证 `coveredStart/EndMessageId` 正确
- 修复手动压缩降级时 `buildMode` 错误设为 `incremental` 的问题：按设计规范保持 `full_rebuild`，通过 `degradeReason` 标记执行层降级
- 修复 `useCompression` 错误消息语言与测试不匹配的问题：统一使用中文
- 修复 `evaluateCompression` 未将当前用户消息纳入体积估算的问题：新增 `currentUserMessage` 参数
- 修复 `parseStructuredSummary` 验证过于宽松的问题：增加数组字段类型校验
- 移除未使用的 `CompressionWindow` 接口死代码
- 修复多段摘要写入原子性问题：多段摘要先以 `draft` 写入，整组成功后再提升为 `valid`
- 修复上下文预算估算重复计入当前用户消息的问题，避免大输入过早触发压缩
- 修复 `useChatStream` 未透传 `modelId` 导致 token 估算路径失效的问题
- 修复多段摘要召回全量注入的问题：按相关性和预算筛选 segment，再注入上下文
- 修复聊天上下文压缩 Phase 2 / Phase 3 设计文档中的审查问题：
  - 为动态 token 压缩阈值补充下界保护，避免小上下文窗口模型出现负阈值
  - 将 system prompt token 缓存改为按 tokenizer/encoding 维度缓存，避免模型切换后复用错误预算
  - 明确文件引用裁剪示例中的 `message_intent` 仍是 message-level intent，并移除误导性的 ref-level 表述
  - 统一 `TokenEstimator` 示例为同步调用，并修复文档中多余的 Markdown 代码块结束标记
- 修复聊天上下文压缩实现中的 token 预算偏差问题：
  - `policy.ts` 新增按模型上下文窗口动态计算的 token 压缩阈值，避免小窗口模型沿用固定 8000 阈值
  - `coordinator.ts` 的预算快照对齐真实发送形状，补计摘要注入与工具定义开销
  - `useChatStream.ts` 透传 providerId 与工具定义到压缩协调器，供预算估算使用
  - `tokenEstimator.ts` 强化消息内容哈希，避免同长度不同内容误复用 token 缓存
- 修复应用启动早期数据库初始化竞态：
  - `src/shared/storage/utils/database.ts` 在 `Database not initialized` 时改为有限重试，减少启动期发送消息被竞态打断
  - `dbSelect` 在重试后仍未就绪时继续只读降级，`dbExecute` 则保留真实失败语义，避免写入假成功
  - `useChatStream.ts` 在读取聊天模型配置时增加启动期重试，避免首次读空导致点击发送静默无响应
  - `index.vue` 在模型配置持续不可用时给出明确错误提示，不再无反馈返回
- 修复聊天侧边栏在存在历史会话但当前未持久化激活会话 ID 时不自动加载数据的问题：
  - `useSession.ts` 初始化阶段会兜底读取最近一条 `assistant` 会话
  - 若找到历史会话则自动恢复该会话 ID，并立即加载对应消息到视图
- 修复全局 SQLite 视图在主进程数据库初始化稍慢时统一读空的问题：
  - `src/shared/storage/utils/database.ts` 将数据库初始化竞态的读写重试窗口从 3 次短重试扩展为更长的有限等待
  - 避免设置页、聊天页等仅在挂载时读取一次的数据视图因为启动抖动而永久停留在空状态
