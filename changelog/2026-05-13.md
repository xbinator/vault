# 2026-05-13

## Changed
- 新增 Tavily 搜索工具设置设计文档，明确 `/settings/tools/search` 的信息架构、配置模型、启用门禁与 `tavilySearch` / `tavilyExtract` 接线方案。
- 将 Electron/Vite 开发环境默认地址统一回 `127.0.0.1`，修复当前环境下 `localhost` 无法解析导致 dev server 启动失败、热更新失效的问题。
- 调整 `TableView` 的 hover overlay 切换策略：新增与删除按钮保持互斥显示，并将切换延时与退场延时抽成可调常量；当鼠标从区段快速切到分割线时，为删除按钮增加延时关闭，缓解频繁切换导致的闪烁问题。
- 将 `TableView` 的 hover overlay 从条件挂载改为 `v-show` 常驻显示切换，并把鼠标跟踪提升到 `viewport` 层，减少悬挂按钮区域的 hover 丢失。
- 修复 `TableView` 外侧悬挂按钮的移入交互：为新增/删除按钮补充命中区域判断，鼠标从表格内容区移向按钮时，不再因为短暂失去表格命中而被提前隐藏。
- 修复 `TableView` 按钮命中区的坐标系错误：将按钮命中框从 `viewport` 局部坐标转换为窗口 `client` 坐标后再比较，避免编辑器不在页面左上角时移向按钮仍被误判为离开。
- 为 `TableView` 补充回归测试，覆盖“区段删除按钮延时关闭后再切换到分割线新增按钮”以及“鼠标移向外侧按钮时保持可见”的交互场景。
- 调整 `TableView` 的新增浮层隐藏策略：隐藏后保留分割线高亮与新增按钮的最后一次定位样式，并为整组 overlay 增加延时退场，避免浮层消失时位置被清空或移向按钮过程中被过早隐藏。
- 调整 `TableView` 的删除浮层缓存策略：隐藏后保留 `segmentHover` 的最后一次行列命中与删除按钮定位样式，避免区段浮层退场后 `segmentHover.value?.row` / `column` 变成 `undefined`。
- 调整 `TableView` 的新增按钮文案：按分割线方向分别显示“新增行”与“新增列”，并同步更新 `aria-label` 以改善可访问性提示。
- 放宽 `TableView` 外侧悬挂按钮的退场缓冲时间，并在鼠标真正进入新增/删除按钮时取消隐藏计时，修复从表格移向按钮过程中控件提前消失的问题。
- 调整 `TableView` 的无命中退场逻辑：鼠标从分割线/区段移向外侧悬挂按钮时，不再立刻清空 overlay，而是进入延迟退场缓冲，避免 `showDividerOverlay` / `showSegmentOverlay` 过早变为 `false`。
- 为 `TableView` 增加全局鼠标轨迹兜底：当指针离开表格命中区但仍停留在“表格边缘到悬挂按钮”的过渡走廊内时，保持新增/删除按钮可点击，并补充对应慢速移入回归测试。
- 修正 `TableView` 全局鼠标保活逻辑对新增按钮的回归影响：保留整组 overlay 的退场缓冲，但不再提前清除“区段切换到分割线”的延时器，避免删除按钮退场过程中把新增按钮的展示状态一并取消。
- 将 `TableView` 的新增控件拆分为行/列双目标：交叉点附近同时展示“新增行”和“新增列”两个按钮，单一分割线附近仅展示对应方向按钮，并补充几何命中与 NodeView 回归测试覆盖双新增场景。
- 修复 `TableView` 双新增按钮的定位偏移：将方向性平移从按钮本体移到外层新增按钮组，避免顶部按钮向右、左侧按钮向上二次偏移，恢复与分割线/边缘的居中对齐。
- 调整 `TableView` 双新增按钮的通道裁剪规则：仅当鼠标同时仍处于顶部与左侧通道时才双显；鼠标移到顶部外侧时隐藏左侧“新增行”，移到左侧外侧时隐藏顶部“新增列”，并补充对应 NodeView 回归测试。
