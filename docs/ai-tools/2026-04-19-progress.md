### Task 04：写工具与确认流程

已完成内容：

- 新增 `src/ai/tools/confirmation.ts`
- 新增写工具：
  - `insert_at_cursor`
  - `replace_selection`
  - `replace_document`
- `src/ai/tools/builtin/index.ts` 现在支持：
  - 默认只返回只读工具
  - 传入 `confirm` 适配器时追加写工具
  - 可选是否包含 `replace_document`

已实现的行为：

- 所有写工具都必须先经过确认。
- 用户取消会返回 `cancelled`，不会直接抛异常。
- `replace_selection` 在没有选区时会返回 `NO_SELECTION`。
- `replace_document` 被视为 `dangerous` 权限。

验证：

- `test/ai/tools/builtin-write.test.ts`
- `test/ai/tools/builtin-read.test.ts`
- `pnpm build`

## 部分完成的任务

### Task 05：聊天侧工具循环与状态机

当前已完成内容：

- 新增 `src/ai/tools/stream.ts`
- 提供 `toTransportTools()`
- 提供 `executeToolCall()`
- 提供 `createToolResultMessages()`
- 在 `src/hooks/useChat.ts` 中接入 `onToolCall`
- 在 `src/components/BChat/index.vue` 中接入：
  - tools 透传
  - tool-call 执行
  - tool result 暂存
  - 第二轮模型续流
  - 工具状态文案
  - 重复 `toolCallId` 防护
  - abort 时本轮工具状态清理
- 在 `src/components/BChatSidebar/index.vue` 中接入：
  - built-in tools 注入
  - 当前 editor context 注入
  - confirm modal 适配器

这意味着：

- renderer 侧已经能接到 `tool-call` 事件
- 已经能把 tool-call 映射到本地工具执行器
- 已经能把内部工具定义转换成 transport tools
- 聊天已经具备“tool-call -> 执行 -> tool result 回注 -> 第二轮继续生成”的主链路

当前仍建议后续补强的部分：

- provider capability gating 做成显式策略，而不是默认信任当前 provider 集合
- 增加更细的聊天层集成测试
- 增加 Electron 手工联调，确认真实工具轮次下的 UI 表现
