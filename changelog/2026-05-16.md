# 2026-05-16

## Changed
- 修复聊天侧边栏 `/compact` 压缩上下文命令的异步派发与压缩输入快照，避免 pending 压缩状态消息进入本次摘要输入。
- 调整 `/compact` 压缩上下文策略，保留最近两轮原文消息作为后续“继续”类指令的上下文锚点。
- 将压缩摘要注入模型的内容升级为结构化上下文块，补充目标、约束、决策、重要事实、文件线索和待处理动作。
- 增加旧压缩记忆的关键词/文件名召回，并在结构化压缩上下文中保留关键工具结果摘要。
- 为 `/compact` 增加重复压缩保护，最新压缩后没有新增对话时不再创建新的压缩任务。
- 将 `useCompactContext` 的消息提示统一切换到 `useInteraction` 交互容器。
- 修正上下文用量在压缩后的空闲态计算，避免继续显示压缩前的旧 API token 用量。
- 增加发送前自动压缩能力，当上下文用量达到动态阈值时先压缩旧消息再发送新请求。
- 修复 `CurrentBlockMenu` 展开方向判断错误，改为在计算时重新定位当前滚动容器边界，避免菜单始终落在默认的 `left-bottom` 布局。
