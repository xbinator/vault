# 2026-04-16

## Changed
- 调整文件重新打开流程：已有最近文件记录时不再先用磁盘内容覆盖本地草稿，避免未显式保存的编辑内容在重新打开时丢失。
- 为最近文件记录补充磁盘同步基线，并在进入编辑器时比对草稿、上次保存内容和当前磁盘内容，只有在可确认安全或用户确认后才刷新为磁盘内容。
- 抽离文件内容冲突判定逻辑并补充自动化测试，覆盖草稿保留、磁盘刷新和已保存基线更新等关键分支。
- 新增独立 `test/` 目录集中管理自动化测试，避免测试文件分散在业务源码目录中。
- 为 `test/` 目录补充 README 约定，统一测试放置方式、命名规则和优先测试的抽象层级。
- 修复历史最近文件记录缺少 `savedContent` 时的兼容问题：重新打开含未保存草稿的本地文件时，会先用磁盘内容补齐保存基线，避免错误清除编辑器 dirty 星号。
