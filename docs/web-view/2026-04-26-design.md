# Tibis WebView 视图技术方案

## 项目背景

- **框架**: Electron 41 + Vue 3.5 + Vite 8
- **现有路由**: 4 个视图模块 (welcome / editor / settings / error)，均使用 lazy-loaded Vue 组件
- **布局**: 单一 default 布局，frameless 窗口 + 自定义标题栏 + Tab 标签页 (KeepAlive 缓存)
- **IPC**: Main process 通过 `contextBridge` 暴露 `electronAPI`，已有一套成熟的 IPC 注册机制
- **现有 webview**: 无，这是全新功能
- **需求**: 浏览外部网页，地址栏输入 URL 导航，轻度交互（获取标题/URL）

---

## 一、实现方式对比

Electron 中嵌入 Web 内容有三种主流方案：

| 方案 | 隔离性 | 灵活性 | Vue 集成 | 推荐场景 |
|---|---|---|---|---|
| `WebContentsView` (Electron 30+) | 独立进程 | 高 | 需 IPC + 主进程管理 | **推荐：新项目首选** |
| `<webview>` 标签 | 独立进程 | 高 | 天然集成 (写在 template 中) | 快速原型 / 简单场景 |
| `<iframe>` | 同进程 | 低 | 天然集成 | 可信内容嵌入 |

> **关键变化**: `BrowserView` 在 Electron 30+ 已被 `WebContentsView` 取代。本项目使用 Electron 41，应使用 `WebContentsView`。

## 二、推荐方案：WebContentsView + Vue 壳组件

### 2.1 架构概览

```
┌──────────────────────────────────────────────────┐
│                   Main Process                    │
│  ┌─────────────────────────────────────┐          │
│  │        WebViewManager               │          │
│  │  - Map<tabId, WebContentsView>      │          │
│  │  - create / destroy / show / hide   │          │
│  │  - loadURL / goBack / goForward     │          │
│  │  - IPC handlers                     │          │
│  └──────────┬──────────────────────────┘          │
│             │ IPC (contextBridge)                  │
├─────────────┼──────────────────────────────────────┤
│  Renderer   │                                      │
│  ┌──────────▼──────────────────────────┐          │
│  │  Vue Shell: views/webview/index.vue │          │
│  │  - 地址栏 (BInput + 回车)            │          │
│  │  - 导航按钮 (后退/前进/刷新)         │          │
│  │  - 页面标题显示                       │          │
│  │  - 加载状态指示器                     │          │
│  │  - 通过 IPC 控制 WebContentsView    │          │
│  └─────────────────────────────────────┘          │
│                                                    │
│  ┌─────────────────────────────────────┐          │
│  │  WebContentsView (独立渲染进程)      │          │
│  │  bounds 由 shell 组件的高度动态计算   │          │
│  └─────────────────────────────────────┘          │
└──────────────────────────────────────────────────┘
```

### 2.2 为什么选 WebContentsView 而非 `<webview>` 标签

| 考量 | WebContentsView | `<webview>` 标签 |
|---|---|---|
| Electron 官方态度 | 推荐，持续维护 | "有已知问题"，不推荐新项目用 |
| 沙箱模式 | 原生支持 `sandbox: true` | 需要额外配置 |
| 进程隔离 | 独立渲染进程，崩溃不影响主窗口 | 独立进程但 API 暴露面大 |
| 与现有 Tab 体系配合 | 通过 IPC 管理显隐 | 依赖 DOM 显隐，与 KeepAlive 有冲突 |
| 安全性 | 由主进程全权控制，最小化攻击面 | 渲染进程 DOM 中暴露完整 webview API |

### 2.3 `<webview>` 标签方案不可行的具体原因

1. **KeepAlive 冲突**: 项目使用 `<KeepAlive>` 缓存视图组件，`<webview>` 标签在组件被缓存/恢复时不会正确暂停/恢复渲染进程，可能导致内存泄漏和僵尸进程
2. **Tab 切换时 layout 问题**: `<webview>` 的 bounds 依赖于 DOM Rect，Tab 切换期间的 CSS display:none 动画会导致 `webview` 的 bounds 计算异常
3. **Electron 官方警告**: `<webview>` 是 Chrome Apps 时代的遗留 API，存在已知的渲染和导航 bug，未来可能被移除

### 3.2 IPC 通道设计

**渲染进程 → 主进程 (invoke/handle)**:

```typescript
// 创建新的 WebContentsView 实例
'webview:create'     (tabId: string, url: string) => void

// 销毁 WebContentsView 实例
'webview:destroy'    (tabId: string) => void

// 导航控制
'webview:navigate'   (tabId: string, url: string) => void
'webview:go-back'    (tabId: string) => void
'webview:go-forward' (tabId: string) => void
'webview:reload'     (tabId: string) => void
'webview:stop'       (tabId: string) => void

// 更新 bounds（当 shell 组件布局变化时）
'webview:set-bounds' (tabId: string, bounds: Rectangle) => void

// 显示/隐藏（Tab 切换时）
'webview:show'       (tabId: string) => void
'webview:hide'       (tabId: string) => void
```

**主进程 → 渲染进程 (事件推送)**:

```typescript
// 页面加载状态变化
'webview:state-changed' (tabId: string, state: WebViewState) => WebViewState

// 页面标题变化
'webview:title-updated' (tabId: string, title: string) => string

// Navigation 状态变化（是否可前进/后退）
'webview:navigation-state-changed' (tabId: string, canGoBack: boolean, canGoForward: boolean)

// 用于在 BrowserView 和主窗口之间同步特定状态
'webview:page-action'    (tabId: string, action: PageAction) => void
```

### 3.3 WebContentsView 生命周期

```
用户点击 Tab / 导航到 /webview
        │
        ▼
  Vue shell mounted (onActivated)
        │
        ├── 调用 IPC 'webview:create' ──► 主进程创建 WebContentsView
        │                                      │
        │                              设置 sandbox + preload
        │                              绑定事件监听器
        │                              webContents.loadURL(url)
        │                                      │
        │                              推送 'webview:state-changed' ──► shell 更新加载状态
        │
        ├── shell 组件计算可用区域 (window size - header - tab bar - address bar)
        │      │
        │      调用 IPC 'webview:set-bounds' ──► 主进程 view.setBounds(...)
        │      │
        │      调用 IPC 'webview:show'
        │
        ▼
  用户切换 Tab / 导航离开
        │
        ├── onDeactivated
        │      │
        │      调用 IPC 'webview:hide' ──► view.setBounds({ x:0, y:0, width:0, height:0 })
        │
        ▼
  用户关闭 Tab
        │
        调用 IPC 'webview:destroy' ──► 主进程移除并销毁 WebContentsView
```

### 3.4 WebViewManager 核心设计

```typescript
/**
 * WebViewManager - 管理所有 WebContentsView 实例
 * 负责创建/销毁、显示/隐藏、bounds 计算、事件监听
 */
class WebViewManager {
  /** 所有活跃的 webview 实例，key 为 tabId */
  private views: Map<string, WebContentsView> = new Map()
  /** 反向映射：webContentsId -> tabId */
  private tabIdMap: Map<number, string> = new Map()
  /** 当前可见的 tabId */
  private activeTabId: string | null = null

  /**
   * 创建新的 WebContentsView
   */
  create(tabId: string, url: string): void {
    const view = new WebContentsView({
      webPreferences: {
        preload: path.join(__dirname, '../preload/webview.mjs'),
        contextIsolation: true,
        nodeIntegration: false,
        sandbox: true,
      }
    })

    // 事件监听：title、load state、navigation、new window
    this.attachListeners(view, tabId)

    // 挂载到主窗口
    const win = BrowserWindow.getAllWindows()[0]
    win.contentView.addChildView(view)
    view.webContents.loadURL(url)

    this.views.set(tabId, view)
    this.tabIdMap.set(view.webContents.id, tabId)
  }

  /**
   * 销毁指定 webview
   */
  destroy(tabId: string): void {
    const view = this.views.get(tabId)
    if (!view) return
    const win = BrowserWindow.getAllWindows()[0]
    win.contentView.removeChildView(view)
    view.webContents.close()
    this.tabIdMap.delete(view.webContents.id)
    this.views.delete(tabId)
  }

  /**
   * 设置当前可见的 webview
   */
  setActive(tabId: string | null): void {
    // 隐藏之前的
    if (this.activeTabId) {
      const prev = this.views.get(this.activeTabId)
      prev?.setBounds({ x: 0, y: 0, width: 0, height: 0 })
    }
    this.activeTabId = tabId
  }
}
```

### 3.5 与现有 Tabs 体系的集成

项目使用 `useTabsStore` (Pinia) 管理标签页，`KeepAlive` 缓存视图。WebContentsView 需要适配这一体系：

1. **Tab 打开**: 路由导航到 `/webview?url=xxx`，创建新 Tab + WebContentsView
2. **Tab 切换**: `onActivated` 时通过 IPC 显示对应 View；`onDeactivated` 时隐藏
3. **Tab 关闭**: 通过 `beforeUnmount` 调用 IPC 销毁 View
4. **同一 URL 复用**: 如果已有相同 URL 的 Tab，切换到该 Tab 而非新建

### 3.6 安全配置

```typescript
// 1. 沙箱模式（必须）
webPreferences: { sandbox: true }

// 2. 拦截新窗口，用系统浏览器打开
view.webContents.setWindowOpenHandler(({ url }) => {
  shell.openExternal(url)
  return { action: 'deny' }
})

// 3. 导航前校验（可选）
view.webContents.on('will-navigate', (event, url) => {
  // 可选：限制允许的域名
  // if (!isAllowedHost(url)) event.preventDefault()
})

// 4. 限制 permissions（禁止摄像头/麦克风/通知等）
session.setPermissionRequestHandler((webContents, permission, callback) => {
  callback(false) // 拒绝所有权限请求
})

// 5. 禁用 devtools（生产环境）
if (!isDev) {
  view.webContents.on('devtools-opened', () => {
    view.webContents.closeDevTools()
  })
}
```

## 五、实施步骤

1. **主进程**: 创建 `electron/main/webview/manager.ts` + `ipc.ts`，实现 WebViewManager
2. **主进程**: 在 `electron/main/index.mts` 中初始化 WebViewManager 并注册 IPC
3. **渲染进程**: 创建 `src/views/webview/index.vue` 壳组件
4. **渲染进程**: 创建 `src/views/webview/hooks/useWebView.ts` 封装 IPC
5. **渲染进程**: 在 `electron/preload/index.mts` 中添加 webview 相关 API
6. **路由**: 添加 `src/router/routes/modules/webview.ts`
7. **类型**: 定义 `src/views/webview/types.ts`
8. **安全**: 配置 sandbox、权限拦截、窗口拦截
9. **集成**: 与现有 Tabs 体系对接（打开/切换/关闭）
10. **测试**: 加载常见网站验证功能

## 六、方案选型总结

| 维度 | 结论 |
|---|---|
| 使用方案 | **WebContentsView**（Electron 41 原生支持） |
| 架构模式 | 主进程管理 View，Vue 渲染壳组件，IPC 通信 |
| 与现有体系集成 | 适配 KeepAlive + TabsStore，通过 onActivated/onDeactivated 控制显隐 |
| 安全性 | sandbox: true + 权限拦截 + 新窗口转系统浏览器 |
| 复杂度 | 中等（需主进程 + 渲染进程协作，但已有成熟的 IPC 基础设施） |
