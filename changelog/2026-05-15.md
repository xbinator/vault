# 2026-05-15

## Changed
- 重写 `write_file` 无工作区相对路径降级设计稿，改为“通用未保存草稿能力 + 严格成功语义”的方案，明确共享工厂转发链路与失败语义。
- 新增并细化 MCP 工具接入设计稿，补充主进程生命周期、discovery 缓存、请求体仅传 server ID、`toolInstructions` 注入规则与配置草稿状态约束。
- 继续收敛 MCP 设计稿，补充请求模型最终态、server-scoped tool 标识、discovery snapshot cache 持久化与并发启动锁约束。
- 将 MCP 设计稿改为“设置页唯一人工配置入口 + BChatSidebar 只读消费 + 模型通过工具调用修改全局配置”的架构，移除聊天侧临时覆盖设计。
- 将 MCP 设计稿升级为基于 `@vercel/sandbox` 的 microVM 执行模型，补充 sandbox 配置字段、`networkPolicy`、snapshot 与运行时分层设计。
- 继续完善 MCP 设计稿，补充 sandbox 依赖容错、IPC 接口、过滤矩阵、cache 精确失效条件、迁移策略与前端状态同步要求。
- 修正 `CurrentBlockMenu` 展开方向判断，改为基于真实滚动容器的可视区域选择 `left-top`、`left-bottom`、`top-left` 或 `bottom`，并在左侧不可用且上方空间充足时新增 `top-left` 回退。
- 在 `CurrentBlockMenu` 类型转换区新增「表格」菜单项，点击后将当前块转换为 3×3 带表头行的表格。
- 为 `CurrentBlockMenu` 补充展开方向回归测试，覆盖“左侧无法容纳面板时回退到底部”和“左侧不可用但上方空间充足时走 `top-left`”的场景。
- 扩展 `BSearchRecent`，支持输入有效绝对路径时显示“按路径打开”候选项，并支持回车优先打开。
- 新增原生路径状态查询能力，用于判断绝对路径是否指向可打开的普通文件。
