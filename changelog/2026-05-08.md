# 2026-05-08

## Changed
- 将聊天侧边栏压缩消息改为时间线式上下文边界节点，不再向用户展示 `recordText`，只保留状态与简短说明
- `/compact` 手动压缩接入聊天发送态，支持通过现有停止按钮中止；中止后压缩消息会持久化标记为 `已取消`
- 将 `handleCompactContext` 从 `index.vue` 抽离到 `hooks/useCompactContext.ts`，收敛手动压缩消息的创建、回填与提示反馈逻辑
- 新增统一聊天任务运行时，聊天发送与 `/compact` 共用底层发送锁、取消和销毁清理语义
- 普通聊天发送移除发送前自动压缩链路，压缩摘要统一改走当前 chat 模型配置
- 收敛压缩运行时命名：`CompressionExecutionResult.summary` 改为 `record`，`useCompactContext` 与相关测试改用 `boundaryText` 等边界语义命名，减少 UI 层对底层 summary 存储概念的泄漏
- 继续收敛压缩模块命名：`previousSummary` 改为 `previousRecord`，`BuildSummaryResult.summaryRecord` 改为 `compressionRecord`，并统一段召回里的 `record` / `maxRecordTokens` 表述
- 将底层压缩记录模型与存储接口彻底迁移为 `compression record` 语义：`summaryText/summaryId` 改为 `recordText/recordId`，`ConversationSummaryRecord` / `SummaryStorage` 改为 `CompressionRecord` / `CompressionRecordStorage`，并同步重命名 SQLite 表与列
- 移除设置页“会话历史压缩助理”相关入口，并从 `ModelServiceType` 中删除 `summarize` 服务类型

## Fixed
- 修复旧版 SQLite 数据库升级后压缩记录表缺失扩展列的问题，避免压缩记录写入时报缺列错误
- 修复 `/compact` 压缩任务中止时 `index.vue` 未透传取消回调，导致停止按钮已复位但压缩消息未立即切换为 `已取消`` 的问题
- 修复消息生成过程中仍可继续发送新消息或触发 `/compact` 的并发穿透问题
- 修复 `handleComplete` 中 `taskRuntime.finishTask` 被 `captureSnapshot` 返回值条件阻断，导致工具调用等待用户输入时 loading 状态卡死的问题
- 修复工具循环续轮期间 `abortActiveTask` 因 `activeTask` 已提前置为 idle 而无法中止流式传输的问题

## Tests
- `chat-storage-sqlite.test.ts`：覆盖旧版 `chat_session_compression_records` 表结构自动补齐压缩记录扩展列的迁移测试
