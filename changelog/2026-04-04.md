- 编辑器：`BEditor` 对外暴露统一搜索接口，页面右侧查找框仅负责输入与触发
- 编辑器：右侧查找栏交互参考 Chrome `Ctrl+F`，补充结果计数、上一个/下一个与关闭操作
- 编辑器：将头部查找栏抽离为独立 `FindBar` 组件，收敛查找 UI 结构与样式
- 编辑器：补充头部工具栏与 `FindBar` 的移动端适配，支持窄屏换行与满宽查找栏
- 编辑器：将查找状态与编辑器搜索联动逻辑内聚回 `FindBar` 组件，移除额外的 `useEditorFind` hook
- 编辑器：恢复缺失的 `extensions/Search` 文件，补齐富文本查找扩展实现与命令声明
- 编辑器：`useEditorFind` 合并重复的 `findKeyword` 监听，抽取正则转义为 `escapeRegExp` 供替换全部复用
- 编辑器：`useEditorFind` 将查找相关 UI 状态收敛为单一 `reactive` 对象，并以 `toRefs` 向模板暴露各字段
- 编辑器：`useEditorFind` 状态字段为 `findOpen` / `keyword` / `matchIndex` / `caseSensitive`，与 `FindBar` 的 `keyword` 模型对齐
- 编辑器：`useEditorFind` 仅对外暴露查找栏所需的 `show` / `keyword` 与操作方法；替换、全量替换、大小写切换等暂不导出；`useEditActive` 不再展开 `editorFind`
- 编辑器：`useEditorFind` 下放到 `FindBar` 内联调用；`index.vue` 只保留 `findBarRef` 与 `openFind` 回调；`useEditActive` 接收 `openFind` 函数
- 编辑器：`BScrollbar` 暴露精确滚动接口，搜索命中项改为通过滚动容器居中定位，减少 `scrollIntoView` 在自定义滚动区域中的偏差
