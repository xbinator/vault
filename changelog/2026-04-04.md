# 2026-04-04

## Changed
- 编辑器：最近文件菜单标签改为优先显示 Markdown 第一条标题，缺失时回退到文件名
- 编辑器：新增 `Search` extension，下沉富文本查找状态、匹配高亮与上下一个跳转逻辑
- 编辑器：`BEditor` 对外暴露统一搜索接口，页面右侧查找框仅负责输入与触发
- 编辑器：右侧查找栏交互参考 Chrome `Ctrl+F`，补充结果计数、上一个/下一个与关闭操作
- 编辑器：将头部查找栏抽离为独立 `FindBar` 组件，收敛查找 UI 结构与样式
- 编辑器：补充头部工具栏与 `FindBar` 的移动端适配，支持窄屏换行与满宽查找栏
- 编辑器：将查找状态与编辑器搜索联动逻辑内聚回 `FindBar` 组件，移除额外的 `useEditorFind` hook
- 编辑器：恢复缺失的 `extensions/Search` 文件，补齐富文本查找扩展实现与命令声明
- 编辑器：`useEditorFind` 合并重复的 `findKeyword` 监听，抽取正则转义为 `escapeRegExp` 供替换全部复用
- 编辑器：`useEditorFind` 将查找相关 UI 状态收敛为单一 `reactive` 对象，并以 `toRefs` 向模板暴露各字段
- 编辑器：`useEditorFind` 状态字段为 `findOpen` / `keyword` / `matchIndex` / `caseSensitive`，与 `FindBar` 的 `keyword` 模型对齐
- 编辑器：`useEditorFind` 仅对外暴露查找栏所需的 `show` / `keyword` 与操作方法；替换、全量替换、大小写切换等暂不导出；`useEditActive` 不再展开 `editorFind`
- 编辑器：`useEditorFind` 下放到 `FindBar` 内联调用；`index.vue` 只保留 `findBarRef` 与 `openFind` 回调；`useEditActive` 接收 `openFind` 函数
- 编辑器：`BScrollbar` 暴露精确滚动接口，搜索命中项改为通过滚动容器居中定位，减少 `scrollIntoView` 在自定义滚动区域中的偏差
- 编辑器：`FindBar` 改为直接读取 `Search` extension 的搜索状态，移除组件内重复的全文匹配计算，简化查找栏代码结构
- 编辑器：补充查找栏交互细节，支持 `Enter` 下一个、`Shift+Enter` 上一个、关闭后恢复编辑器焦点，并在打开时自动聚焦输入框
- 编辑器：恢复富文本模式下丢失的当前块菜单挂载与组件实现，重新接通块类型切换、插入和块级操作入口
- 编辑器：修复当前块菜单点击前因编辑器失焦被卸载的问题，改为在 `mousedown` 阶段接管触发与菜单项交互
- 编辑器：当前块菜单面板改用 `BScrollbar` 承载，并限制最大滚动高度，统一滚动体验与样式
- 编辑器：调整当前块菜单面板展开方向，改为从触发按钮左侧弹出，避免遮挡正文区域
- 编辑器：当前块菜单在代码块和表格相关节点上自动隐藏，避免在不适合的块类型上显示通用块操作
- 编辑器：当前块菜单隐藏逻辑补充编辑器上下文判断，修复代码块内单行选中时菜单仍然显示的问题
- 编辑器：修复当前块菜单在光标位于块末尾时误归属到下一个元素的问题，边界命中优先归到前一个块
- 编辑器：收紧当前块边界判断，仅在块起始边界时回退到前一位置，避免块首与块尾场景混淆
- 编辑器：当前块定位补充鼠标命中位置参与判断，改善光标处于块边界时菜单归属不稳定的问题
- 编辑器：当前块菜单隐藏逻辑补充 DOM 容器检测，在 `pre` 和表格容器内统一隐藏，避免代码块容器内仍然显示菜单
- 编辑器：修复鼠标从编辑器移入当前块菜单触发按钮时菜单消失的问题，新增菜单自身悬停态参与显隐判断
- 编辑器：收敛当前块菜单的鼠标命中更新时机，菜单打开或悬停时不再被编辑器 `mousemove` 持续改写当前块
- 编辑器：当前块菜单改为基于鼠标命中的实际 DOM 块元素定位，替代 `posAtCoords` 推断，减少边界和嵌套节点场景下的抖动
- 编辑器：当前块菜单触发按钮改为展示当前块类型图标，不再固定使用加号图标
- 编辑器：当前块菜单触发图标改为基于鼠标命中的当前块类型计算，不再依赖编辑器当前选区状态
- 编辑器：当前块菜单显示与定位改为仅基于鼠标命中的块，不再使用光标选区作为兜底来源
- 编辑器：修复 `CurrentBlockMenu` 卸载时重复访问已销毁编辑器视图导致的 `beforeUnmount` 报错，清理阶段改为使用缓存的编辑器 DOM 引用
