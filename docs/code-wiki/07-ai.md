# 07-AI 模块

## 总体设计

AI 模块分为“渲染进程调用层”与“主进程执行层”：

- 渲染进程：负责从设置中解析 provider 配置、发起 invoke/stream、处理流式事件回调
- 主进程：负责模型创建、调用第三方 SDK、统一错误格式并把结果回传给渲染进程

## 渲染进程：useAgent

- `src/hooks/useAgent.ts`
- 输入：`providerId`（MaybeRefOrGetter），回调 `onChunk/onError/onComplete`
- 核心步骤：
  1. `providerStorage.getProvider(providerId)` 解析 provider 详情
  2. 生成 `AICreateOptions`（providerType/apiKey/baseUrl 等）
  3. 非流式：`electronAPI.aiInvoke(createOptions, request)`
  4. 流式：
     - 订阅 `onAiStreamChunk/Complete/Error`
     - 调用 `electronAPI.aiStream(createOptions, request)` 启动主进程流式任务
     - 完成/失败后清理所有订阅

## 主进程：AIService

- `electron/main/modules/ai/service.mts`
- 关键成员：
  - `aiProvider: AIProviderRegistry`：provider registry（把 providerType + modelId 映射为可用的 SDK model）
- 关键方法：
  - `generateText(createOptions, request)`：调用 `ai.generateText`
  - `streamText(createOptions, request)`：调用 `ai.streamText`，返回 `textStream`
- 错误处理：
  - 捕获 `unknown` 错误，统一交给 `AIProviderRegistry.normalizeError(error, providerType)` 转换成可序列化的错误对象（用于 IPC 传递）

## Provider Registry

- `electron/main/modules/ai/providers/_index.mts`
- 职责：
  - 按 providerType 注册 provider 工厂（openai/anthropic/google）
  - `create(createOptions, modelId)`：返回符合 ai SDK 要求的 model 实例
  - `normalizeError(error, providerType)`：按 provider 做错误归一化

## IPC：invoke 与 stream

- `electron/main/modules/ai/ipc.mts`
  - `ai:invoke`：直接 `return await aiService.generateText(...)`
  - `ai:stream`：
    - 找到当前窗口 `getWindowFromWebContents(event.sender)`
    - `for await (const chunk of result.stream)`：逐 chunk `win.webContents.send('ai:stream:chunk', chunk)`
    - 完成：`ai:stream:complete`
    - 失败：`ai:stream:error`（发送已归一化的 error 对象）

## 数据与类型

- AI 相关类型放在 `types/ai.d.ts`
  - `AICreateOptions`：providerType/providerName/providerId/apiKey/baseUrl
  - `AIRequestOptions`：modelId/prompt/system/temperature 等
  - `AIInvokeResult`：text + usage（token 统计）
  - `AIServiceError`：主进程归一化后的错误格式（包含 code/message 等）
