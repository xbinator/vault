# 2026-04-23

## Added
- 新增 `ask_user_choice` 内置工具的阶段性实现，包含等待用户输入结果类型、执行器、默认工具目录注册和执行器校验测试。
- 补充 `ask_user_choice` 后续实现事项文档，记录剩余集成、交互和验证工作。
- 新增聊天用户选择卡片，支持 `ask_user_choice` 的单选、多选和其他手动输入提交。
- 新增 `update_settings` 内置工具，支持 AI 经用户确认后修改主题、大纲、源码模式和侧边栏相关设置。
- 新增 AI 工具权限模式与授权记忆，支持 `update_settings` 在只读、自动安全和用户授权场景下按策略执行。
- 新增聊天侧边栏历史消息基于时间戳游标的分页加载能力。
- 新增 macOS Dock 菜单和 Windows 任务栏 Jump List 快捷入口，支持新建、打开最近列表和直接打开最近文件。

## Changed
- 忽略项目本地 `.worktrees/` 目录，避免 Git worktree 内容进入仓库状态。
- 完成 `ask_user_choice` stream 与 BChat 集成，等待态会暂停工具续轮，用户提交答案后通过工具结果通道继续生成。
- 调整 AI 主进程限流和过载错误日志，避免后台标题生成失败时输出冗长堆栈。
- 移除默认内置工具中的 `get_current_selection` 选区读取能力。
- 修复 `get_current_time` 被误判为需要活动编辑器文档的问题，使其可在无文档时执行。
- 修复 `ask_user_choice` 被误判为需要活动编辑器文档的问题，使无文档场景也可向用户提问。
- 调整 AI 工具执行器上下文校验，允许 `update_settings` 等全局工具在没有活动编辑器文档时执行。
- 将内置工具权限字段从 `permission` 迁移为 `riskLevel`，明确区分工具风险等级和用户授权策略。
- 调整聊天消息读取逻辑，首次加载仅取最新一段，滚动到历史边界前再按 `createdAt + id` 复合游标加载更早消息。
- 调整聊天侧边栏会话切换和新建会话流程，流式输出期间禁用会话切换、删除和新建会话操作。
- 将最近文件同步收口到文件 Store，由渲染进程只同步 `id/name/path` 摘要给系统快捷入口。
