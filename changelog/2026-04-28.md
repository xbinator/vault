# 2026-04-28

## Changed
- SelectionToolbar: BubbleMenu 关闭策略重构 —— BubbleMenu 通过 `shouldShow` 控制显隐，点击"AI 助手"/"插入对话"后锁死当前选区范围，相同选区不重复弹出；用户变更选区后自动解锁，恢复正常弹窗
- SelectionToolbar: 编辑器失焦后菜单不再关闭，聚焦到编辑器面板内非编辑区（如 FrontMatterCard）时正常消失，外部区域保持显示
- BPromptEditor: `cm-widgetBuffer` 宽度改为随内容空态切换，空内容保持 `1px`，有内容时收缩为 `0`
- Docs: 最近文件方案从"补 `touch` 调用点"重构为长期架构设计，明确时间语义、统一打开用例和派生排序规则
- Docs: 新增“最近文件按 openedAt 排序”实施计划，拆分存储层、files store、打开入口、保存路径与测试任务，便于后续按阶段执行
- Recent Files: `StoredFile` 新增 `createdAt/openedAt/modifiedAt/savedAt` 及预留字段，兼容旧数据缺失并补齐存储注释
- Recent Files: 存储层改为按 `openedAt -> modifiedAt -> createdAt` 派生排序，新增 `touchRecentFile()` 与串行写队列，避免并发读改写覆盖
- Recent Files: `files` store 新增统一打开用例，欢迎页、搜索、菜单、原生打开与拖拽入口统一改为通过 store action 更新最近文件顺序
- Recent Files: 编辑器保存链路只更新 `savedAt` 与内容相关时间字段，不再通过保存操作影响 `openedAt` 排序
- Tests: 修复 Windows 下 `pnpm test` 的脚本兼容性问题，引入 `cross-env`，并补充 recent storage / files store 回归测试
- Tests: 同步 BEditor、BChat、BChatSidebar 与 router 的过期源码断言到当前实现，避免路径重构和等价写法差异导致的误报
